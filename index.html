Lo que hacía antes

Typora es genial porque me permite usar markdown para escribir archivos grandiosos (y por lo tanto tomar notas) muy fácilmente.

Anteriormente estaba realizando este proceso usando vscode para escribir el .md (el cual me permite una visualización al lado del texto que voy escribiendo) y luego usando el terminal, usar pandoc para importar ese documento a .pdf.

Ese método es bueno, pero no el mejor, porque tengo que escribir en un lado, ver la previsualización de markdown al lado (el cual no muestra las ecuaciones latex) y luego sacar el terminal, compilar con

pandoc input.md -o output.pdf

cada vez que quiero ver el pdf. Esto, incluso teniendo la extensión de vscode para ver pdf, es tedioso...

Ahora uso Typora

Typora es un editor minimalista, con las cosas necesarias, muy fácil de instalar y con muchos temas disponibles. Me permite hacer todo lo que mencioné con VScode, puedo escribir súper rápido en markdown pero al mismo tiempo el editor va formateando de tal forma que veo de una vez como se ve el texto en markdown (cuenta con una vista en código por si quiero ver literalmente lo que vería en un archivo de texto), adicionalmente, si quiero insertar puedo hacerlo usando lo que ya conozco dentro de $$ (obviamente cosas básicas, si quiero algo muy complejo, lo mejor es exportar a .Tex y luego trabajar con eso dentro de TexStudio por ejemplo)

Formas de Compilar

  1. Directamente desde Typora se puede exportar a muchos formatos como .pdfy HTML y queda bonito pero digamos que no tan 'Latexoso', pero si es muy eficiente y no toca hacer casi que nada. Creo que de por si Typora puede exportar a esos dos formatos sin ayuda de pandoc

  2. La otra forma es usando pandoc y mikTex.

    • Instalar pandoc y mikTex.
    • Una vez escrito el archivo .md en Typora o VScode o literalmente donde sea, incluso en el bloc de notas,
    • Abrimos la terminal de windows (cmd) en donde se encuentre el archivo. (esto puede ser así nada más, o si se quiere abrir en vscode la carpeta en donde se encuentra el archivo como workspace y de ahí usar la terminal integrada es otra opcion)
    • usamos el comando pandoc input.md -o output.pdf donde input es el nombre del archivo de entrada y output el nombre que queremos para el archivo de salida.

Esto crea un archivo más 'Latexoso' pero requiere de más trabajo. Sin embargo, vale la pena conocer esta forma, porque puede ser útil en otros contextos, por ejemplo al usar Jupyter-Lab es posible crear un archivo MarkDown como pestaña adicional y luego correr usando la terminal integrada (también en otra pestaña) el comando visto usando pandoc. (Aún así es mucho mejor usar Typora en una ventana aparte y no complicarse tanto)

Instalación de Pandoc y Miktex

En ambos casos es suficiente con buscar por internet y descargar del primer resultado (las páginas oficiales) los ejecutables para la instalación. Así de sencillo. Adicionalmente, se puede instalar TexStudio también de la web oficial.

Preferencias de Typora

  • Usar temas oscuros (para descargar más a parte del que viene por defecto ir al sitio web de Typora, a la pestaña de temas y al final dan una guía de instalación)

  • Activar para que se vean bonitas las ecuaciones en el editor.

    • Opción archivo->preferencias->markdown->ecuacion en linea

  • Cambiar formato el formato de exportación en pdf para una salida en tamaño carta y no A4.

  • Activar Autoguardado y opción de completar pares de simbolos como ** `` {} ...

  • Activar en exportar la opción de ver la carpeta a la cual se exportó el documento.
